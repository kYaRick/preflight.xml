## Що налаштовує ця секція

Windows Setup читає з `autounattend.xml` три параметри локалізації й застосовує їх
на різних етапах інсталяції:

- **Мова інтерфейсу** &mdash; мова самого Windows (Параметри, Провідник, сповіщення).
  Виводиться в компоненті `Microsoft-Windows-International-Core-WinPE`, щоб UI інсталятора
  з'являвся одразу вибраною мовою, і ще раз у `Microsoft-Windows-International-Core` —
  для фінальної сесії користувача.
- **Мова вводу** &mdash; розкладка клавіатури на першому завантаженні. Винесена окремо,
  бо користувач у Німеччині часто хоче англійський інтерфейс і німецьку клавіатуру.
- **Країна / регіон** &mdash; код країни, що впливає на форматування (дати, валюта, одиниці)
  й регіонально-залежні сервіси (як-от каталог Microsoft Store). В XML перетворюється на
  числовий "geo ID".

Всі три мають розумні дефолти, що повторюють вихід schneegans-генератора — тож найпоширеніший
випадок (англійський інтерфейс, американська клавіатура, США як регіон) працює без правок.

## Поради

- **Мова інтерфейсу** визначає, які мовні пакети Windows очікує знайти в образі. Якщо
  обрати мову, якої в образі немає — система автоматично впаде до англійської.
- **Мова вводу** приймає просту BCP-47 позначку (`en-US`, `uk-UA`) для типових кейсів;
  складніші сценарії (основна + додаткова розкладки, кастомний KLID) задаються через
  розширений синтаксис `InputLocale` — див. посилання на MS Learn біля відповідної опції.

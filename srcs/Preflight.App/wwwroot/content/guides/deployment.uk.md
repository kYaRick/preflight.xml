# 📄 Як використати свій `autounattend.xml`

Ти згенерував файл автоматизації встановлення. Сам по собі він нічого не робить - програма встановлення Windows має знайти його під час інсталяції. Тут показано, куди його покласти.

## 🔍 Як Setup знаходить файл

Під час автоматичного встановлення Windows Setup самостійно шукає файл з іменем **точно** `autounattend.xml` у **корені** кожного доступного диска, зокрема на знімних USB-носіях. Спрацьовує перший збіг. Тож зазвичай задача проста: *покласти `autounattend.xml` у корінь диска, під'єднаного під час роботи Setup.*

Два моменти, на яких часто спотикаються:

- Ім'я має бути саме `autounattend.xml` - не `unattend.xml`, не `autounattend (1).xml`.
- Файл має лежати в **корені** диска (`E:\autounattend.xml`), а не в підпапці.

---

## 🔌 Спосіб 1 - Окрема USB-флешка (найпростіше, ISO не чіпаємо)

Працює з *будь-яким* завантажувальним носієм: DVD з Windows, флешкою з Rufus, диском Ventoy, навіть мережевим (PXE) завантаженням. Інсталяційний носій узагалі не змінюється.

1. Візьми будь-яку вільну USB-флешку. Відформатуй у **FAT32**.
2. Скопіюй `autounattend.xml` у її **корінь**.
3. Завантаж цільовий ПК зі звичайного інсталяційного носія Windows, маючи цю флешку під'єднаною.
4. Setup знаходить файл і запускається автоматично.

> Флешка може бути маленькою й дешевою - важливий лише цей один файл. Мітка тому значення не має.

---

## 🛠️ Спосіб 2 - На самій інсталяційній USB (Rufus)

Якщо робиш інсталяційну флешку через [Rufus](https://rufus.ie):

1. Запиши ISO Windows на USB через Rufus як завжди.
2. Коли запис завершиться - відкрий цю флешку в Провіднику.
3. Скопіюй `autounattend.xml` у її **корінь** (поруч із `setup.exe`).
4. Завантажся з неї.

Новіші версії Rufus також пропонують *власне* вікно кастомізації («прибрати вимоги / створити локальний акаунт»). Це окреме від твого файлу - пропусти його й використовуй власний `autounattend.xml`, інакше Rufus може підсунути конкурентний файл і виникне конфлікт.

---

## 🧰 Спосіб 3 - Ventoy

Два варіанти, обери один.

**A. Окрема флешка (найпростіше).** Ventoy завантажує ISO, а Setup усе одно сканує всі диски. Тож просто виконай **Спосіб 1** - поклади `autounattend.xml` на будь-яку іншу FAT32-флешку, під'єднану під час інсталяції. У самому Ventoy налаштовувати нічого не треба.

**B. Плагін Auto-Installation у Ventoy (один диск, повторюваний результат).**

1. Скопіюй `autounattend.xml` кудись на дата-розділ Ventoy, наприклад `\templates\autounattend.xml`.
2. Створи або відредагуй `\ventoy\ventoy.json` на цьому розділі:

```json
   {
     "auto_install": [
       {
         "image": "/ISO/Win11.iso",
         "template": "/templates/autounattend.xml"
       }
     ]
   }
```

   - `image` - шлях до твого ISO **на диску Ventoy** (з початковим `/`).
   - `template` - шлях до файлу автоматизації на тому ж диску.
3. Завантаж Ventoy, обери ISO. Ventoy підставляє шаблон, і Setup іде автоматично.

Прожени `ventoy.json` через [валідатор JSON](https://jsonlint.com) - зайва кома тихо вимикає плагін.

## 💿 Спосіб 4 - Вшити всередину ISO (один самодостатній образ)

Корисно для архівування або там, де не можна додати другий диск (деякі VM, деякі PXE-сетапи). Розпаковуєш ISO, додаєш файл, перепаковуєш як завантажувальний.

**Через GUI (найпростіше):** відкрий ISO у програмі, що вміє перезбирати завантажувальні образи - **AnyBurn**, **UltraISO** або **NTLite** - додай `autounattend.xml` у корінь і збережи як новий ISO. Не пошкодь завантажувальний запис.

**Через командний рядок (`oscdimg`, з Windows ADK):**

1. Встанови компонент *Deployment Tools* з [Windows ADK](https://learn.microsoft.com/windows-hardware/get-started/adk-install).
2. Розпакуй вміст ISO у папку, наприклад `C:\iso`.
3. Скопіюй `autounattend.xml` у `C:\iso\autounattend.xml` (корінь).
4. Перепакуй у завантажувальний UEFI+BIOS ISO:

```powershell
   oscdimg -m -o -u2 -udfver102 `
     -bootdata:2#p0,e,bC:\iso\boot\etfsboot.com#pEF,e,bC:\iso\efi\microsoft\boot\efisys.bin `
     C:\iso C:\Windows-unattended.iso
```

   Запускай з ярлика *Deployment and Imaging Tools Environment*, щоб `oscdimg` був у PATH.

## 🖥️ Спосіб 5 - Віртуальні машини

У VM немає «другої USB-флешки», тож під'єднай файл як маленький диск:

- **Найпростіше:** зроби крихітний ISO лише з `autounattend.xml` у корені (будь-яким ISO-інструментом або командою `oscdimg -n -m C:\xmlfolder C:\answer.iso`), потім під'єднай його як **другий** DVD-привід поруч з інсталяційним ISO Windows. Setup його просканує.
- Або під'єднай маленький **віртуальний диск FAT32** із файлом у корені.
- Hyper-V / VirtualBox / VMware працюють однаково - файл просто має бути на *якомусь* диску під час інсталяції.

Для готового образу VM **Спосіб 4** (вшити в ISO) повністю прибирає потребу в другому диску.

## ✅ Перевірка, що спрацювало

- Якщо Setup зупиняється й питає мову, редакцію, розділи чи ім'я користувача - файл **не** знайдено або **не** прочитано. Перевір ім'я та що він лежить у корені.
- Логи Setup розташовані в `X:\Windows\Panther\` під час інсталяції та `C:\Windows\Panther\` після неї. Файли `setupact.log` / `setuperr.log` фіксують, який файл автоматизації підхопився та які були помилки розбору.

## ⚠️ Перед запуском на реальній машині

Якщо твій конфіг витирає диск (Disk → автоматичний режим / wipe), інсталяція **зітре цільовий диск без жодного запиту**. Спершу завжди прожени на віртуальній машині й двічі перевір, який диск обере Setup, перш ніж розгортати на залізі.
